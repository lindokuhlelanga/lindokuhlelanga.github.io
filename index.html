FM Transmitter & Receiver System

Custom RF Communication with Digital Signal Processing

2024
Embedded Systems, RF
Electronics Project

Project Overview

Designed and built a complete FM radio communication system from scratch, including both transmitter and receiver circuits. The project demonstrates deep understanding of RF (Radio Frequency) electronics, analog circuit design, and digital signal processing principles.

Key Features

  • Custom-designed FM transmitter operating in the 88-108 MHz frequency band with adjustable carrier frequency
  • Frequency modulation circuit with audio input processing and pre-emphasis for improved signal-to-noise ratio
  • RF amplification stage providing 100mW output power with efficient power management
  • Superheterodyne receiver architecture with intermediate frequency (IF) amplification at 10.7 MHz
  • Automatic frequency control (AFC) for stable signal reception and frequency locking
  • Audio amplification circuit with volume control and de-emphasis for balanced frequency response
  • Power supply design with voltage regulation ensuring clean DC power for sensitive RF components
  • PCB layout optimized for RF performance with proper impedance matching and minimal interference

Technical Implementation

The transmitter uses a voltage-controlled oscillator (VCO) with varactor diode for frequency modulation. The audio signal is conditioned through pre-emphasis filtering before modulating the carrier. The receiver employs a superheterodyne design with local oscillator, mixer, IF amplifier, and discriminator for demodulation. All circuits were prototyped on breadboard, tested, and then transferred to custom PCB designs.

Circuit Design Details

  • Oscillator circuit using Colpitts configuration for stable frequency generation
  • RF amplifier with Class C operation for maximum efficiency
  • Impedance matching networks using LC circuits for optimal power transfer
  • Antenna design with quarter-wave monopole for efficient radiation
  • Noise filtering using capacitors and RF chokes at strategic points

Results & Impact

Successfully achieved wireless audio transmission over 50 meters with clear audio quality and minimal distortion. The project required extensive troubleshooting and optimization of RF circuits, providing valuable hands-on experience with analog electronics and electromagnetic principles. Demonstrated ability to work with complex analog systems and apply theoretical knowledge to practical implementations.

Network Traffic Engineering & Optimization

Advanced Optimization using Graph Theory and Linear Programming on GEANT2 Topology

September - November 2025
Networking, Optimization
Network Systems 3 (NSS370S)

Project Overview

Developed a comprehensive network traffic engineering solution using mathematical optimization techniques on the GEANT2 topology - a real-world European research and education network with 32 nodes (major cities) and 54 bidirectional links. The project demonstrates mastery of graph theory, linear programming, and network optimization algorithms to achieve efficient traffic routing while minimizing congestion and cost.

Key Features

  • Multi-Model Optimization: Implemented three distinct optimization approaches - Min-Cost Flow (cost minimization), Min-Max Utilization (load balancing), and Multi-Commodity Flow (simultaneous multi-flow routing)
  • Dijkstra's Algorithm: Computed shortest paths with O((V+E)logV) complexity for efficient route discovery between Zagreb (node 21) and Barcelona (node 25)
  • Linear Programming Formulation: Developed complete mathematical models with flow conservation constraints, capacity limits, and non-negativity conditions
  • Single-Commodity Flow (SCF): Optimized 250 Mbps traffic flow achieving 10% maximum utilization on shortest path (cost = 4,000)
  • Multi-Commodity Flow (MCF): Simultaneously routed 5 traffic demands totaling 1,000 Mbps (100, 150, 200, 250, 300 Mbps) with 14% maximum utilization
  • Load Balancing: Min-Max model distributed traffic across 26 links achieving 0.83% maximum utilization for network resilience
  • Statistical Analysis: Comprehensive performance metrics including mean utilization (4.57%), standard deviation (4.06%), and bottleneck identification
  • Network Visualization: Created detailed topology diagrams, shortest path visualizations, and utilization heatmaps using NetworkX and Matplotlib

Mathematical Framework

The project applies rigorous mathematical formulations to network optimization problems. The core framework models the network as a directed graph G=(V,E) where V represents nodes (cities/routers) and E represents links with associated capacities and weights. Each optimization model solves a constrained linear program:

  • Decision Variables: f_ij^k representing flow of commodity k on link (i,j)
  • Objective Functions: Minimize total cost c_ijf_ij or minimize maximum utilization
  • Flow Conservation: Ensures data in equals data out at each intermediate node (Kirchhoff's law analogy)
  • Capacity Constraints: Total flow on each link cannot exceed its capacity (f_ij^k c_ij)
  • Non-negativity: All flow variables must be positive (f_ij^k 0)

Technical Implementation

The solution architecture leverages Python's scientific computing ecosystem. NetworkX library provides graph data structures and built-in algorithms including Dijkstra's shortest path. PuLP linear programming library formulates and solves optimization problems with CBC solver backend. The implementation includes automated demand generation based on student number, topology creation with 32 nodes and 54 edges, and comprehensive result visualization using Matplotlib.

Algorithm Details

  • Dijkstra's Shortest Path: Priority queue-based algorithm computing minimum-cost paths with O((V+E)logV) time complexity
  • Min-Cost Flow: Linear program minimizing w_ijf_ij subject to flow conservation and capacity constraints
  • Min-Max Utilization: Minimizes maximum link load () ensuring balanced network utilization and resilience
  • Multi-Commodity Flow: Extends single-flow optimization to K simultaneous demands with shared link capacities
  • Capacity Allocation: Ring topology with 2,500 Mbps base capacity, cross-links with 10,000 Mbps for long-distance connections

Optimization Results & Analysis

All optimization models achieved optimal solutions without demand scaling (scaling factor = 1.0). The shortest path from Zagreb to Barcelona was computed as 21202625 with total cost of 16 and 3 hops. SCF Min-Cost model achieved lowest total cost (4,000) but concentrated traffic on 3 links with 10% max utilization. SCF Min-Max model distributed load across 26 links achieving excellent balance (0.83% max utilization). MCF model successfully routed 5 commodities simultaneously with moderate cost (21,550) and 14% maximum utilization across 14 active links.

Statistical Findings

  • Network Efficiency: GEANT2 topology is well-provisioned with maximum utilization only 14% under full load
  • Bottleneck Analysis: Ljubljana-Bratislava link identified as most utilized (10% in SCF) due to lower capacity (2,500 Mbps)
  • Load Distribution: Min-Cost shows skewed load (variance 4.33%), Min-Max shows uniform distribution (variance 0.28%)
  • Path Diversity: GEANT2's ring-and-cross topology provides sufficient redundancy with 54 links for 32 nodes
  • Quartile Analysis: Median utilization 3.0% (MCF), 2.5% (SCF Min-Cost), 0.83% (SCF Min-Max)

Cost vs. Resilience Trade-off

The project clearly demonstrates the fundamental trade-off in network engineering between cost optimization and resilience. Min-Cost routing achieves lowest operational cost but creates vulnerability by concentrating traffic on shortest paths - a single link failure could disrupt entire flows. Min-Max routing ensures robust operation through balanced load distribution but requires more active links and higher total cost. The MCF model offers a practical middle ground, achieving reasonable cost while maintaining moderate load distribution suitable for production networks.

Real-World Applications

  • OSPF Routing: Similar to Min-Cost approach using Dijkstra's algorithm for shortest-path routing in enterprise networks
  • MPLS Traffic Engineering: Comparable to Min-Max approach providing load-balanced, constraint-based routing
  • SDN Controllers: Modern software-defined networks (Google B4, Microsoft SWAN) use MCF-style optimization for dynamic routing
  • Internet Backbone: ISPs apply traffic engineering to balance loads across diverse paths and prevent congestion

Results & Impact

Successfully delivered a production-grade network optimization framework demonstrating deep understanding of graph algorithms, linear programming, and network engineering principles. The project received excellent academic feedback for mathematical rigor, comprehensive documentation, and practical applicability. All models achieved optimal convergence confirming correct constraint formulation and solver configuration. The analysis provides actionable insights for real-world network operators balancing cost, performance, and resilience requirements.
